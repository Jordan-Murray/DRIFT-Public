How DRIFT Works

Last updated: 9 April 2026

DRIFT is a Jira app for requirements traceability, backlog review, and drift detection. It helps teams ingest a source requirement baseline, link Jira issues to those requirements, review mismatches before delivery, and keep a human decision trail for backlog changes.

What the app does

DRIFT adds two main surfaces inside Jira:

  • Project page: manage requirements, review coverage, and generate Jira stories from requirements
  • Issue panel: link an issue to one requirement, validate alignment, and record a review decision

How DRIFT works

  • Import or paste source requirements from text, documents, or Confluence
  • Parse those requirements into a project-level baseline stored in the app
  • Link Jira issues to one primary requirement
  • Validate issue content against the linked or suggested requirement
  • Show findings, AI-assisted review output when configured, and human review states
  • Track requirement coverage based on explicitly linked Jira issues
  • Optionally draft Jira stories directly from uncovered requirements

Key features

  • Requirement ingestion from pasted text, uploaded files, and Confluence pages
  • Deterministic validation for timing, numeric, scope, and coverage mismatches
  • Optional AI semantic review using OpenAI or Anthropic when configured
  • AI-assisted requirement extraction from messy or loosely structured source text
  • Explicit issue-to-requirement linking
  • Review workflow with pending, approved, intentional deviation, and false positive states
  • Requirement coverage tracking based on linked Jira issues and Done status
  • Draft-story generation from a requirement with editable acceptance criteria

Setup and usage

Typical usage inside Jira:

  • Install DRIFT into Jira Cloud
  • Open the DRIFT project page in the target Jira project
  • Paste requirements or import them from a document or Confluence page
  • Review the parsed requirements list and project coverage summary
  • Open a Jira issue and use the DRIFT issue panel to link it to one requirement
  • Run validation and review the findings
  • Approve the issue, mark it as an intentional deviation, or mark it as a false positive

What reviewers and testers should expect

DRIFT is designed for teams that want a quality gate between source requirements and Jira delivery work. The app is most useful when stories are drafted from formal requirements, client documents, Confluence pages, or AI-assisted backlog generation workflows.

Suggested Marketplace test flow

  1. Open the DRIFT project page in a Jira project.
  2. Paste a small requirement set such as: REQ-1: Users must be able to export audit logs in CSV format only.
  3. Save or ingest the requirements.
  4. Create or open a Jira issue with conflicting wording such as Allow export in any format.
  5. Open the DRIFT issue panel and link the issue to the requirement.
  6. Run validation and confirm that DRIFT flags the mismatch.
  7. Record a review decision such as pending, approved, or false positive.
  8. Return to the project page and confirm the queue and coverage state update.

Configuration notes

  • AI-assisted features are optional and require provider keys to be configured by the operator
  • Confluence import requires access to the target Confluence page on the same Atlassian site
  • Coverage is based on explicit requirement links, not fuzzy suggestions
  • DRIFT outputs support human review and should not be treated as automatic delivery approval
